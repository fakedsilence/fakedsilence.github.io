开发日志_第一周

发表于2024-10-13|更新于2024-10-13
|阅读量:

开发日志（10.7~10.13）

程序

  • 使用 Git 管理游戏资源，提升了版本管理和团队协作的效率。
  • 编写了 Luna 的移动脚本，实现了流畅的角色控制。
  • 增加了 Ethan 的跟随机制，使他能够在 Luna 移动时自动跟随，增加了伙伴互动的真实感。

美术

  • 绘制了侦探社的场景，丰富的室内陈设和氛围营造。
  • 完成了主要人物头像的设计，包括 Luna 和 Ethan，力求在视觉上体现他们各自的性格特点。

一些思考

  • 放弃了大地图设计，改为在固定地点的探索模式，例如第一章集中在教学楼内进行探索。这样设计可以提高玩家的目标感，使他们更清楚自己当前的探索任务和位置。通过在特定区域设置多个场景，玩家能够明确当前任务的探索区域，不再在大地图中迷失方向。同时，限定场景的设计更便于营造紧张和压抑的氛围。例如，教学楼内不同楼层的探索，可以逐步加深恐怖感，让玩家在心理上逐步感受到压迫。

  • 策划案中新增的灵体交锋需要进一步设计，目前的想法是在适当的地方新增不友善灵体，对Luna的情感弱点（内疚、恐惧）进行攻击。玩家需要选择适当的对话来稳定Luna的情绪。负面效果目前的想法是低语、游戏画面扭曲的效果。此外，稳定Luna的情绪暂定使用一个钟摆（呼应侦探室内的布置，可以展开一些故事情节），每当钟摆摆动的时候，速度由快到慢，范围逐渐缩小，玩家需要越来越集中精力，把握每一次钟摆的落点。

  • 从开发的角度来看，缩小探索范围可以在一个较小的区域内精雕细琢，提供更高质量的视觉和互动体验，同时降低开发难度。目前游戏开发的首要目标是完成作品。

  • 是否使用 Unity 插件 Dialog System ，虽然这个插件功能强大，但是否需要这么复杂的游戏剧情复杂度和演出效果，还需要进一步考虑。
